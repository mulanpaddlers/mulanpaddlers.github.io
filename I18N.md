# Internationalization (i18n)

## Languages

* English (en)
* Simplified Chinese (zh)

## Routing

* /en
* /zh

## Rules

* NO hardcoded UI strings in components
* All text must come from translation files
* Keep meaning consistent, not word-for-word literal
* Default locale: en

## Structure

/messages/en.json
/messages/zh.json

## Navigation

* Language toggle must preserve current route
* Visible in header

## Formatting

* Dates/numbers: keep simple (no heavy localization in v1)

## Content Flow

CONTENT.md → converted into messages JSON for runtime
